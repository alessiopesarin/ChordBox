## 📌 Project Overview
This project trains a neural network to perform Automatic Chord Recognition (ACR) from raw audio. 
The core philosophy relies on **Semi-Supervised Learning**, specifically the **Iterative Self-Training (Noisy Student)** paradigm. The model is initially trained on high-quality, human-annotated datasets, and then used as a "Teacher" to generate pseudo-labels for a massive corpus of unlabeled YouTube audio. A "Student" model is then trained on these pseudo-labels with added noise to surpass the Teacher's performance.

## 🏗️ Core Architecture (The SOTA)
The project now supports several architectures selectable interactively:

*   **BASELINE** (`ChordCRNN`): 2-layer CNN + Bi-LSTM. Lightweight and efficient.
*   **DEEP** (`DeepChordCRNN`): 4-layer CNN + 3-layer Bi-LSTM. Higher expressive capacity.
*   **ENHANCED** (`EnhancedChordCRNN` - **Current SOTA**): Residual CNN + Bi-LSTM + Self-Attention + LayerNorm.

## 📂 Directory Structure & File Roles

### 📦 Data Organization (Raw vs Processed)
We have reorganized the project to follow data cleanliness principles:
*   `data/raw/`: Contains original datasets (Billboard, GuitarSet, Synthetic) and CSV metadata.
*   `data/processed/`: Contains precomputed features (CQT tensors `.pt`) ready for training.

### 🚀 Training & Orchestration
*   `train.py`: Now includes an interactive interface to choose which model to train or start sequential training for all architectures (**Train All**).
*   `src/training/trainer.py`: Manages the training loop with strict isolation of historical records for each architecture.
*   `src/web/main.py`: Entry point for the FastAPI Web App (moved to keep the root clean).
*   `outputs.md`: Automatic log of all performance results obtained after each training session.

### Legacy / Alternative Architectures (Do NOT use unless explicitly asked)
* `MultiTaskModel.py`: Attempted predicting Root (13 classes) + Quality (6 classes). Good root accuracy, but low strict accuracy.
* `PreTrainedCRNNModel.py` / `extract_vggish.py`: Attempted using Google's VGGish as a feature extractor. **Failed approach**: VGGish destroys fine-grained harmonic frequencies needed for chords.
* `StudentModel.py` / `CRNNModel.py`: Older, purely sequential architectures (pre-ResNet/Attention).

### Data & Datasets
* `models/`: Directory containing saved `.pth` weights. Models track their *own* validation loss historically (e.g., `student_gen_1_1.4984.pth`).
* **Clean Datasets (Ground Truth):**
    * `billboard_audio/` -> `billboard_features_pt/`: McGill Billboard dataset (CQT precomputed).
    * `GuitarSet_Dataset/` -> `guitarset_features_pt/`: GuitarSet dataset (CQT precomputed, labels parsed directly via pure JSON to avoid `mirdata` overhead).
* **Unlabeled Data (YouTube / Noisy Student):**
    * `util/dataset_venticello/`: Raw MP3s downloaded from YouTube.
    * `util/youtube_features_pt/`: Precomputed and normalized CQT tensors for YouTube audio.
    * `venticello.csv`: Playlist metadata.
    * `venticello_pseudo_labels.csv`: The output database generated by the Teacher, containing `filename`, `frame_idx`, `chord`, and `confidence`.

### Utility Scripts (`util/`)
* `download_playlist.py` / `download.py`: Scripts utilizing `yt-dlp` to download audio safely. Idempotent (skips existing files).
* `precompute_youtube.py` / `precompute_guitarset.py`: Critical scripts to convert raw audio into `.pt` CQT tensors *before* training. They enforce strict CQT normalization: `(chroma - mean) / std`.

## 🧠 Strategic Lessons Learned (Do not revert these decisions)
1.  **CQT over Mel/VGGish:** Chords require exact frequency bins. Mel-spectrograms and VGGish embeddings blur this data. We strictly use CQT with `hop_length=512`, `sr=22050`.
2.  **CQT Normalization is Mandatory:** If `generate_pseudo_label.py` or the `Dataset` class forgets to normalize the CQT exactly as done in training, the model will output garbage predictions.
3.  **Attention OOM:** Passing a full 3-minute song to Self-Attention causes `CUDA Out of Memory`. Inference must be done in chunks (e.g., 430 frames / ~10 seconds) and stitched back together.
4.  **Student Promotion:** A Noisy Student only saves a new file if it beats its *own* past epochs, not the Teacher's initial high score. It must be promoted to Teacher only if it converges successfully.
